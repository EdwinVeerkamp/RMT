• Wofür?

    Die Website unterstützt Fußball-Teams bei der Verwaltung von Spielen und Trainings.

    Der Team-Manager stellt Termine ein und versendet Einladungen, auf die die Spieler antworten können. Der Manager kann so jederzeit feststellen ob genug Spieler für den Termin zurückgemeldet sind.

  • Warum?

    Das Projekt ist eine Fortführung eines alten PHP-Skripts, das mittlerweile in die Jahre gekommen ist. Neben der Bespaßung der Benutzer dient das Projekt dem Autor als Spielwiese für die Evaluierung und den Einsatz von Web-Technologien.

  • Beta?

    Beta bedeutet:
    • Darstellung optimiert für Firefox.
      Andere Browser (insb. Internet Explorer) können Bereiche der Website z.T. nicht korrekt darstellen.
    • Features kommen und gehen
    • Fehler können jederzeit auftreten
    • Die Website kann jederzeit offline sein

  • Hosting

    • Provider: Host Europe
    • Server: Virtual Server Windows 2000 2 GB RAM
    • Servlet-Container: Tomcat (behind IIS with IIS-Tomcat Connector)
    • Datenbank: MySQL

  • Technik

    Das Projekt basiert auf folgendem Java Technologie-Stack:
    • Wicket
    • Spring Security
    • Spring
    • Spring Data JPA
    • Hibernate
    • Hibernate Validator
    • AspectJ
    • Velocity
    User interface:
    • Twitter Bootstrap
    • Less
    • JQuery

  • Tools

    Für die Entwicklung werden verwendet:
    • IDE: IntelliJ IDEA
    • Datenbank: H2
    • Testing: TestNG, Mockito, Scala
    • Build: Gradle
    • Continous Integration: Jenkins
    • Version Control System: Git
    • Github Repo: https://github.com/flowerrrr/RMT
    • Project Management: Acunote (Web-based Scrum Tool)
    • Design: Gliffy (Web-based Wireframes)
